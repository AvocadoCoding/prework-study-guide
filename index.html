HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • In the head element information like the character set and viewport can be specified.
  • h1,h2,h3 etc. are heading types.
  • Commands like break, paragraph, line, list and section can be used to order page content.
  • Semantic HTML elements indicate the role of their content, as well as giving sturcture to webpage.

CSS

  • Three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, external CSS style sheet.
  • Inline CSS is applied to spcific HTML tags, using a style attribute with CSS rule for a particular element.
  • An entire stylesheet with rules for the webpage can also be directly embedded using a style element in the head element.
  • An external CSS style sheet that is lined in the head element is generally preferred - seperation of concerns.
  • Seperation of concerns is based on the idea that each section of code should have its own responsibility.
  • A CSS rule has selector defining where the rule will apply, and a declaration which the property to apply and it's value.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class attribute allow us to identify different elements, to apply different rules to elements with the same name.
  • We can style HTML elements using the elements as selectors or by using the class attribute as a selector.

Git

  • Can push and pull code from gitHub.
  • Use SSH key to provide easy access to gitHub project.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • Do itmes of work on feature branch and then merge with man branch.
  • Don't make too much changer per revision, in case it needs to be undone during debugging later.

JavaScript

  • Javascript files are linked into HTML files in the body element.
  • Javascript files give a webpage functionality.
  • The console.log method was created to make it easier for developers to test their code.
  • Use Chrome Dev Tools to debug code and view console.log information. To access 'right click' and then select 'inspect'.
  • JS is read from top to bottom and the order counts.
  • Falsy values in JS are 0, -0, On, "", null, undefined, NaN or a Boolean data type with the value of false
